# 屏幕伴侣插件更新日志

## 2.0.0 (2026-03-05)

### 重大更新
- **框架人格支持**：现在可以使用 AstrBot 框架的默认人格了，插件会优先使用 `persona_manager` 获取人格，其次是配置文件中的 `system_prompt`
- **人格统一**：所有 LLM 调用（开始回复、结束回复、屏幕分析、日记生成、日记偷看回复）现在都统一使用相同的人格设定
- **超时优化**：
  - 视觉 API 调用超时从 60 秒增加到 120 秒
  - 开始/结束回复超时从 30 秒增加到 60 秒
  - 屏幕分析超时保持 120 秒
- **日记图片优化**：
  - 图片宽度从 600px 增加到 800px
  - 字体大小从默认增加到 20px
  - 改进了中文字符的换行算法
  - 使用 `font.getlength()` 替代 `font.getbbox()` 获得更准确的文字宽度
- **任务取消修复**：
  - 修复了 `kps` 命令无法正确结束 task_0 的问题
  - 改进了异步任务取消处理，正确传播 `asyncio.CancelledError`
- **提示词优化**：
  - 简化了所有提示词，去掉了语气规定
  - 保持简短，尽量在一句话内说完
  - 日记总结保持在 400 字左右

### 配置更新
- 新增 `system_prompt` 配置项，用于自定义人格设定
- 统一了配置默认值，确保与代码中的默认值一致

### 修复
- 修复了回复没有应用框架人格的问题
- 修复了日记图片因为宽度不足被截断的问题
- 修复了视觉 API 调用超时问题
- 修复了 LLM 生成结束回复超时问题
- 修复了不同回复的提示词被错误地合在一起的问题

## 1.9.0 (2026-03-04)

### 新增功能
- **调试模式**：添加了调试模式开关，可通过 `/kpi debug [on/off]` 命令控制，关闭时隐藏大部分日志
- **日记补写**：添加了 `/kpi complete [YYYY-MM-DD]` 命令，可以补写指定日期的日记

### 改进
- **简化命令**：简化了 KPI 子指令，将 `/kpi recent-diaries` 简化为 `/kpi recent`，将 `/kpi diary-completion` 简化为 `/kpi complete`
- **平台适配**：修复了 l2d 桌宠平台的主动消息发送问题，确保消息能够正确发送到桌宠客户端
- **日志优化**：更新了日志系统，使其在调试模式关闭时更加简洁

### 修复
- 修复了主动消息发送目标的问题，确保使用配置的 `proactive_target`
- 修复了 l2d 平台下没有主动消息的问题

- 自定义监控任务：添加了定时监控任务功能

### 改进

- 环境感知：添加了系统状态、天气和节假日感知
- 互动模式：添加了多种预设互动模式

### 修复
- 修复了截图失败的问题
- 修复了视觉API调用失败的问题
