# 屏幕伙伴更新日志

## 2.7.0 (2026-03-17)

### 调整

- 更新插件与 WebUI 版本号到 `2.7.0`。
- 重写说明文档，整理安装、配置、排障和 WebUI 使用说明。
- 精简重复指令，对外只保留简化版入口：
  - `/kpi status`
  - `/kpi p`
  - `/kpi d`
  - `/kpi cd`

### 优化

- 录屏识屏新增轻量采样与自适应采样策略，优先抽关键帧后再决定是否进入完整视频分析。
- 日记升级为“双层产物”，除自然语言正文外还会生成结构化摘要。
- 主动消息补充更细的打扰控制，包括相似回复冷却、同窗口频率限制和手动发言后暂缓打断。
- WebUI 支持查看更可解释的识屏信息，包括触发原因、素材类型、识别摘要与最终回复。
- 长期记忆与活动统计继续补强，提升持久化、检索质量和可观察性。

## 2.6.2 (2026-03-17)

### 调整

- 修正午夜自动日记的日期归属。
- `/kpi cd` 在凌晨两点前默认补写前一天的日记。
- 优化 `/kpr` 的过程反馈与录屏链路超时策略，降低录屏模式下的超时概率。

## 2.6.1 (2026-03-16)

### 新增

- 新增 `/kpi ffmpeg [路径]` 指令，可自动复制 `ffmpeg.exe` 到插件目录。
- 新增录屏模式配置校验，避免检查间隔小于录屏时长。

### 调整

- `/kpr` 现在始终执行一次录屏识别，不再受全局识别素材开关影响。
- 优化观察记录去重逻辑，保留更多有效观察。
- `recording_fps` 支持小数。

## 2.6.0 (2026-03-15)

### 新增

- 新增截图/录屏识别素材开关。
- 新增录屏识别链路与 `/kpr` 指令。
- 新增 Gemini 原生多模态直连优化。
- 新增 `ffmpeg_path`、`recording_fps` 和 `recording_duration_seconds` 配置项。
- WebUI 增加最新素材预览。

### 调整

- `use_external_vision` 明确区分“外部视觉 API 两步识别”和“直接发送给 AstrBot 的多模态识别”。
- `/kp` 固定为截图识别，`/kpr` 固定为录屏识别。
- 录屏默认参数调整为 `1 fps / 10 秒`。
